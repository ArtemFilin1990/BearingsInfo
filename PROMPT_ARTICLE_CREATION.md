# Промт для создания отдельных статей по каждому подразделу технического справочника о подшипниках

## Основная задача

Создать полностью самостоятельные статьи для каждого пункта указанного оглавления, чтобы при переходе по ссылке пользователь попадал на конкретную статью с исчерпывающей информацией по этой теме.

---

## Требования к структуре каждой статьи

### 1. Заголовок и метаданные

```markdown
# [Номер и полное название подраздела]
*Дата последнего обновления: 2026-01-10*
*Ключевые слова: [список ключевых терминов через запятую]*
```

**Пример:**
```markdown
# 1.1. Что такое подшипник? Основные функции
*Дата последнего обновления: 2026-01-10*
*Ключевые слова: подшипник качения, трение качения, тела качения, дорожка качения, функции подшипника*
```

---

### 2. Навигация внутри статьи (в начале)

```markdown
## Содержание
- [Раздел 1](#якорь-1)
- [Раздел 2](#якорь-2)
- [Раздел 3](#якорь-3)
- [Ключевые выводы](#выводы)
- [Связанные материалы](#связанные)
```

**Примечание:** Якоря создаются автоматически из заголовков разделов (все буквы в нижнем регистре, пробелы заменяются на дефисы).

---

### 4. Обязательные элементы в каждой статье

#### 4.1. Таблицы

Каждая статья должна содержать **минимум 2 таблицы**:
- Сравнительная таблица (различных типов, характеристик, параметров)
- Справочная таблица (размеров, аналогов, технических данных)

**Формат таблицы:**
```markdown
| Параметр | Вариант 1 | Вариант 2 | Вариант 3 | Примечание |
|----------|-----------|-----------|-----------|------------|
| Характеристика А | Значение | Значение | Значение | Пояснение |
| Характеристика Б | Значение | Значение | Значение | Пояснение |
| Характеристика В | Значение | Значение | Значение | Пояснение |
```

**Пример сравнительной таблицы:**
```markdown
| Характеристика | Подшипники качения | Подшипники скольжения | Примечание |
|---|---|---|---|
| Коэффициент трения | 0,001-0,005 | 0,01-0,10 | В 10-100 раз меньше |
| Момент пуска | Низкий | Высокий | Требуется преодоление трения покоя |
| Габариты | Больше в радиальном направлении | Компактнее | Зависит от конструкции |
| Срок службы | Ограничен усталостью материала | Определяется износом | При правильной эксплуатации |
```

#### 4.2. Примеры

Каждая статья должна включать:
- ✅ Пример расчета (если тема предполагает вычисления)
- ✅ Пример маркировки с пошаговой расшифровкой
- ✅ Пример подбора для конкретных условий эксплуатации

**Формат примера маркировки:**
```markdown
### Пример расшифровки: 6205-2RS

**Пошаговая расшифровка:**
1. **6** — тип подшипника (шариковый радиальный однорядный)
2. **2** — серия диаметров (лёгкая серия)
3. **05** — внутренний диаметр: 05 × 5 = 25 мм
4. **2RS** — суффикс: уплотнения с двух сторон (резиновые)

**Полная расшифровка:**
Шариковый радиальный однорядный подшипник лёгкой серии с внутренним диаметром 25 мм и резиновыми уплотнениями с обеих сторон.

**Источник:** ГОСТ 3478-2012
```

#### 4.3. Визуальные элементы

Укажите места для необходимых визуальных материалов:

```markdown
[Схема: Принцип работы подшипника качения]
[График: Зависимость срока службы от нагрузки]
[Фото: Типичный вид подшипника данного типа]
[Диаграмма: Классификация подшипников по типу]
```

**Примечание:** Визуальные элементы должны быть размещены в папке `Изображения/` с понятными названиями файлов.

### 5. Раздел "Ключевые выводы"

В конце каждой статьи обязателен блок с основными выводами:

```markdown
## Ключевые выводы {#выводы}

1. **Вывод 1** - краткая формулировка основного тезиса
2. **Вывод 2** - краткая формулировка второго важного момента
3. **Вывод 3** - краткая формулировка третьего ключевого аспекта
4. **Вывод 4** - практическая рекомендация (при необходимости)
```

**Пример:**
```markdown
## Ключевые выводы {#выводы}

1. **Подшипники качения** - основной тип опор в современном машиностроении, обеспечивающий минимальное трение и высокую энергоэффективность
2. **Принцип работы** основан на замене трения скольжения трением качения, что снижает коэффициент трения в 10-100 раз
3. **Основные функции** включают обеспечение свободного вращения, точное позиционирование вала и восприятие радиальных и осевых нагрузок
4. **Области применения** охватывают практически все отрасли промышленности - от бытовой техники до авиакосмической отрасли
```

## 7. Технические требования к контенту

### Объем статьи

| Тип статьи | Минимум | Оптимально | Максимум |
|------------|---------|------------|----------|
| Теоретическая (Часть I) | 1500 слов | 2000-3000 слов | 5000 слов |
| Справочная (Части II-IV) | 1500 слов | 2000-2500 слов | 4000 слов |
| Стандарты и нормативы | 1000 слов | 1500-2000 слов | 3000 слов |

### Глубина проработки

- ✅ Дайте ответы на типичные вопросы по теме
- ✅ Рассмотрите все аспекты, указанные в заголовке
- ✅ Приведите данные для различных условий эксплуатации
- ✅ Укажите ограничения и особенности применения
- ✅ Обеспечьте техническую точность всех утверждений
- ✅ Приведите ссылки на источники (ГОСТ, ISO, каталоги производителей)

### Стиль изложения

- **Технически точный**, но понятный широкой аудитории
- **Структурированный** с четкой логикой и последовательностью
- **Практико-ориентированный** с конкретными примерами
- **Без "воды"** - только полезная и проверенная информация
- **Согласованный** с существующей терминологией в репозитории

## 9. Формат вывода

### Именование файлов

Каждая статья должна быть сохранена как отдельный Markdown-файл с именем, соответствующим номеру и теме подраздела:

**Формат имени файла:**
```
[номер]-[краткое-название-латиницей].md
```

**Примеры:**
- `1-1-chto-takoe-podshipnik.md` - для раздела "1.1. Что такое подшипник?"
- `2-3-oboznachenie-razmernyh-serij.md` - для раздела "2.3. Обозначение размерных серий"
- `8-2-iso-standarty.md` - для раздела "8.2. ISO (международные стандарты)"
- `11-1-salniki-i-manzhety.md` - для раздела "11.1. Сальники и манжеты"

### Структура директорий

```
/home/runner/work/Baza/Baza/
```

- ❌ Информация без указания источника
- ❌ Приблизительные или непроверенные данные
- ❌ Противоречивая информация из разных стандартов в одной статье

### Терминология

- Используйте единую терминологию во всех статьях
- При первом упоминании термина давайте его определение
- Поддерживайте словарь терминов в разделе "Приложения"
- Указывайте английские эквиваленты технических терминов

### Перекрестные ссылки

- Активно используйте ссылки на другие статьи
- Создавайте логические связи между разделами
- Избегайте дублирования информации - лучше дать ссылку
- Проверяйте работоспособность всех ссылок

## Заключение

Этот промт гарантирует, что каждая созданная статья будет:

1. **Самостоятельной** - содержит всю необходимую информацию по теме
2. **Структурированной** - имеет четкую логическую структуру
3. **Практичной** - содержит конкретные данные, примеры и таблицы
4. **Связанной** - имеет навигацию к другим статьям и ресурсам
5. **Полной** - раскрывает тему максимально глубоко и всесторонне
6. **Проверенной** - опирается на официальные источники и стандарты

При соблюдении всех требований этого промта вы создадите качественный технический справочник, который будет полезен как начинающим специалистам, так и опытным инженерам.
