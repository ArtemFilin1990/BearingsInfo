# Roadmap энциклопедии BearingsInfo (Baza)

**Дата создания:** 2026-01-21  
**Версия:** 1.0  
**Статус:** В разработке

---

## О документе

Этот документ содержит базовый список тем для энциклопедии BearingsInfo (Baza). Roadmap покрывает обозначения, стандарты, конструкцию, эксплуатацию и сопоставления ГОСТ ⇄ ISO.

**Назначение:**
- Систематизировать знания о подшипниках в репозитории
- Обеспечить структурированное наполнение энциклопедии
- Гарантировать полноту охвата тем
- Создать единый стандарт для статей

**Принципы наполнения:**
1. Строгая структура статей (см. [PROMPT_ENCYCLOPEDIA_ASSISTANT.md](PROMPT_ENCYCLOPEDIA_ASSISTANT.md))
2. Обязательная таблица ГОСТ ⇄ ISO для всех тем
3. Трассируемость к файлам репозитория
4. Проверяемость фактов
5. Отсутствие домыслов — только данные из репозитория

## Как работать дальше

### Варианты стратегии наполнения:

#### Вариант 1: Последовательное наполнение
Идти по порядку разделов 1→2→3→...→14 и превращать каждый пункт в полноценную энциклопедическую страницу.

**Преимущества:**
- Систематичность
- Полнота охвата
- Последовательное накопление знаний

**Недостатки:**
- Долго до первых результатов
- Может быть неравномерная ценность разделов

#### Вариант 2: Приоритетное наполнение
Выбрать критически важные темы и сфокусироваться на них:

**Критический приоритет (делать первыми):**
1. Система обозначений ГОСТ и ISO (раздел 2)
2. Сравнение обозначений ГОСТ ⇄ ISO (раздел 2.7)
3. Таблица аналогов ГОСТ ⇄ ISO (раздел 12.2)
4. Основные стандарты (ГОСТ 520, ISO 15, ISO 492)
5. Грузоподъёмность и расчёт ресурса (разделы 6.6, 11)

**Преимущества:**
- Быстрая практическая ценность
- Фокус на главном
- Гибкость планирования

**Недостатки:**
- Может оставить пробелы в некритичных темах

#### Вариант 3: Табличный фокус
Начать с тем, где критически важны таблицы ГОСТ ⇄ ISO:

**Темы с табличным приоритетом:**
1. Соответствие стандартов (раздел 3.6)
2. Сравнение обозначений (раздел 2.7)
3. Классы точности (раздел 4.3)
4. Классы зазоров (разделы 5.3, 5.4)
5. Аналоги подшипников (раздел 12.2)

**Преимущества:**
- Максимальная структурированность
- Сразу видна полнота/неполнота данных
- Легко проверяемо

**Недостатки:**
- Требует качественных источников данных в репо
- Может быть сложно найти все соответствия
