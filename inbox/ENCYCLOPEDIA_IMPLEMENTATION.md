## Ключевые особенности системы

### 1. Строгая трассируемость

**Проблема:** Невозможно проверить источник информации  
**Решение:** Обязательный раздел "Файлы-источники" в каждой статье

**Пример:**
```markdown
## 0. Файлы-источники

- `data/csv/master_catalog.csv` — таблица аналогов ГОСТ ⇄ ISO
- `03_ГОСТ_подшипники_и_нормативка/ГОСТ_520.md` — технические условия
```

### 2. Обязательная таблица ГОСТ ⇄ ISO

**Проблема:** Путаница в соответствиях стандартов  
**Решение:** Таблица присутствует в КАЖДОЙ статье, даже если "Не применимо"

**Форматы:**
- Полное соответствие
- Частичное соответствие (с указанием различий)
- Отсутствие аналога
- Спорное соответствие

### 3. Запрет внешних источников

**Проблема:** Неконтролируемые данные извне  
**Решение:** Source of truth = только файлы репозитория

**Последствия:**
- Если данных нет в репо → явно указывается "Данные отсутствуют"
- Создаётся раздел TBD с указанием, что нужно добавить в репо
- Обеспечивается воспроизводимость и проверяемость

### 4. Запрет длинных фраз в таблицах

**Проблема:** Таблицы превращаются в "простыни" текста  
**Решение:** Tables policy — только короткие значения

**Разрешено:**
- Числа: `3...1600 мм`
- Обозначения: `ГОСТ 520-2011`
- Короткие фразы: `Действующий`

**Запрещено:**
- Длинные предложения
- Абзацы текста
- Развёрнутые описания

### 5. Система TBD (To Be Done)

**Проблема:** Неясно, какие данные отсутствуют в репо  
**Решение:** Раздел 12 явно указывает, что нужно добавить

**Формат:**
| Что нужно | Зачем | Где взять/как проверить |
|-----------|-------|------------------------|
| Полный ГОСТ 520-2011 | Проверить формулировки | Добавить PDF в `03_ГОСТ.../` |

## Примеры использования

### Сценарий 1: Создание новой статьи

**Задача:** Создать статью "Радиальные шарикоподшипники"

**Шаги:**
1. Открыть ENCYCLOPEDIA_INDEX.md
2. Найти тему в разделе 1.1
3. Скопировать _template-article.md
4. Изучить PROMPT_ENCYCLOPEDIA_ASSISTANT.md
5. Собрать данные из репо:
   - `02_Термины_и_основы/Теория/типы_подшипников/`
   - `data/csv/master_catalog.csv`
6. Заполнить все 12 разделов
7. Создать таблицу ГОСТ ⇄ ISO (раздел 4)
8. Указать файлы-источники (раздел 0)
9. Заполнить TBD (раздел 12)
10. Проверить по Done Criteria

### Сценарий 2: Обновление существующей статьи

**Задача:** Добавить трассируемость к существующей статье

**Шаги:**
1. Проверить структуру по PROMPT_ENCYCLOPEDIA_ASSISTANT.md
2. Добавить раздел 0 "Файлы-источники"
3. Проверить/создать раздел 4 "ГОСТ ⇄ ISO"
4. Добавить раздел 12 "TBD"
5. В разделе 10 разделить источники на:
   - Файлы из репо
   - Упоминаемые стандарты
6. Проверить отсутствие внешних ссылок

### Сценарий 3: Планирование наполнения

**Задача:** Определить приоритеты на ближайший месяц

**Шаги:**
1. Открыть ROADMAP_ENCYCLOPEDIA.md
2. Найти темы с приоритетом "КРИТИЧЕСКИЙ"
3. Выбрать Фазу 1 (Недели 1-2):
   - Система обозначений ГОСТ
   - Система обозначений ISO
   - Сравнение ГОСТ ⇄ ISO
4. Проверить источники данных в репо:
   - `05_Маркировка_суффиксы_серии/`
   - `04_ISO_и_международные_обозначения/`
5. Создать статьи по шаблону
