# ГОСТ база данных (машиночитаемая)

## Что здесь
- `data/gost/bearings.csv` — типы и описания обозначений (6205, 7805, 180205, 7205, 7505).
- `data/gost/dimensions.csv` — размеры d/D/B, приведённые из таблиц размеров.
- `data/gost/series.csv` — сериальные признаки (серия ширины/диаметров) из примеров 6205 и 7805.
- `data/gost/tolerances.csv` — классы точности 0/6/5/4/2.
- Расшифровки и примеры берутся из `sources/gost/ПОДШИПНИКИ ГОСТ.docx` и сопутствующих стандартов.

## Как читать обозначения
1. **Тип**: первая цифра задаёт класс подшипника (6 — радиальный шариковый, 7 — конический).
2. **Серия диаметров/ширины**: в обозначении 6205 серия диаметров = 2, серия ширины = 0; у 7805 серия диаметров = 8.
3. **Код отверстия**: 05 → d = 25 мм, по правилу ГОСТ/ISO (00–03 = 10/12/15/17 мм, 04–96 = код × 5 мм).

## Примеры применения
- **6205**: радиальный шариковый, серия 2/0, d = 25 мм. Аналоги и размеры лежат в `data/analogs/gost_iso.csv` и `data/gost/dimensions.csv`.
- **7805**: конический роликовый, серия 8/0, требует регулировки преднатяга; размеры в `data/gost/dimensions.csv`.
- **2205/22205/NU 205**: при замене цилиндрического на сферический нужно контролировать ширину (B) и зазор — см. записи в `data/analogs/gost_iso.csv`.

## Проверка данных
- Уникальные ключи: `designation` в bearings/tolerances/series и `(designation, d, D, B)` в dimensions.
- Сортировка фиксирована по обозначениям и размерам; изменять порядок строк руками запрещено — используйте `python scripts/update_repo.py`.
- Числа записываются с точкой и без пробелов, пустые значения оставляются пустыми строками.

## Обновление
```bash
python scripts/update_repo.py         # нормализация CSV
python scripts/validate/run_validations.py  # проверки схем
```
Отчёты о сборке формируются в `data/reports/YYYY-MM-DD_source.json` и фиксируют входные файлы из `sources/gost/*`.
